Routes & security groups 12
Two layers, two jobs

Route tables are signposts. Security groups are doormen.

Route table
destination          target
─────────────────    ─────────────

0.0.0.0/0            nat-09f8e7   ← private subnet

For traffic to this destination, send to this target.

Security group

Security groups are stateful — return traffic for an allowed inbound connection is implicit.

Two layers, two jobs. Routes decide where; security groups decide who. Stateful matters: you don't write a return rule.
