What IAM is 35
Identity & Access Management

IAM is the doorman, not the data.

IAM controls who can do what in your AWS account. It doesn't store application data.

Every AWS request — console click or API call — passes through IAM.

Every AWS request answers a question:
is this principal allowed to do this action on this resource?
scope IAM is global — no Region selection.
The frame: IAM is an authorization service, not a data store. Every console click and API call passes through it.
